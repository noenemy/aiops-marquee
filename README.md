# AIOps Typography Rotation

AWS Summit 데모 부스 운영을 위한 풀스크린 타이포그래피 로테이션 디스플레이.

스탠드형 TV에 AIOps 홍보 문구를 자동으로 순환 표시한다. 빌드 도구 없이 정적 파일과 로컬 웹 서버만으로 동작하며, `messages.txt` 파일을 수정하는 것만으로 표시 문구를 즉시 교체할 수 있다.

## 주요 기능

- **풀스크린 타이포그래피 카드**: 메인 헤드라인 + 서브 텍스트 구조, 1920×1080 TV에 최적화된 대형 폰트
- **자동 로테이션**: 8초 간격으로 카드 순환, 마지막 카드 이후 첫 번째로 무한 반복
- **부드러운 페이드 전환**: CSS opacity 기반 페이드인/아웃 (총 1초 이내)
- **카드별 시각적 다양성**: 7가지 AWS 브랜드 컬러 그라데이션이 카드마다 변형되어 적용
- **AWS Summit 브랜딩**: 좌측 상단에 AWS 로고와 "AWS SUMMIT SEOUL 2026" 헤더 고정
- **실시간 문구 갱신**: `messages.txt`를 저장하면 3초 안에 화면에 자동 반영 (서버 재시작 불필요)
- **오프라인 동작**: 외부 네트워크 의존성 없음, 시스템 폰트 스택 사용
- **장시간 안정성**: 글로벌 에러 핸들러 + 메모리 누수 방지로 8시간 이상 연속 운영 가능

## 실행 방법

### 사전 요구사항

- Python 3 (macOS/Linux는 기본 설치됨)
- 모던 웹브라우저 (Chrome, Edge, Safari, Firefox 최신 버전)

### 1. 로컬 서버 시작

```bash
./serve.sh
```

기본 포트는 `8000`이며, 다른 포트를 사용하려면 인자를 전달한다.

```bash
./serve.sh 8765
```

### 2. 브라우저로 접속

```
http://localhost:8000
```

데모 부스 TV에서 사용할 때는 브라우저를 풀스크린 모드(F11 또는 macOS의 Ctrl+Cmd+F)로 띄운다.

### 3. 종료

서버를 띄운 터미널에서 `Ctrl+C`.

## 홍보 문구 편집

`messages.txt` 파일을 텍스트 에디터로 열어 수정한다. 형식 규칙은 다음과 같다.

- **빈 줄**로 메시지를 구분한다
- **첫 번째 줄**: 메인 텍스트 (큰 폰트)
- **두 번째 줄**: 서브 텍스트 (보조 폰트, 선택)

예시:

```
AI가 장애를 예측합니다
사후 대응에서 사전 예방으로

운영 비용을 40% 절감
지능형 리소스 최적화

24/7 자동 모니터링
사람이 놓치는 이상 징후를 AI가 감지
```

저장하면 약 3초 이내에 화면에 자동 반영된다(서버나 브라우저 재시작 불필요). 파일 로드에 실패하거나 비어있을 경우 내장된 기본 메시지 세트로 자동 폴백된다.

## 파일 구성

| 파일 | 설명 |
|------|------|
| `index.html` | 풀스크린 레이아웃과 헤더, 카드 컨테이너 |
| `style.css` | AWS 브랜드 컬러 디자인 토큰, 타이포그래피, 페이드 전환 |
| `app.js` | 6개 모듈 통합 (파싱·로테이션·전환·스타일·파일감시·에러처리) |
| `messages.txt` | 표시할 홍보 문구 데이터 |
| `aws-logo.png` | 화면 좌측 상단 AWS 로고 |
| `serve.sh` | 로컬 웹 서버 실행 스크립트 |
| `.kiro/specs/aiops-typography-rotation/` | 요구사항·설계·태스크 스펙 문서 |

## 모듈 구조

`app.js`는 IIFE namespace 패턴으로 6개 모듈을 `window.App`에 노출한다.

- `App.MessageParser` — `messages.txt` 텍스트를 메시지 배열로 파싱, 기본 폴백 문구 보유
- `App.RotationEngine` — 8초 간격 로테이션 상태/타이머 관리 (setTimeout 체인)
- `App.TransitionManager` — 페이드인/아웃 전환 (Promise 기반)
- `App.StyleManager` — 카드별 그라데이션·강조 색상 팔레트
- `App.FileWatcher` — `messages.txt` 변경 감지 (3초 간격 polling)
- `App.ErrorHandler` — 글로벌 에러 핸들러 + 로테이션 자동 재시작

## 커스터마이징

| 변경하고 싶은 것 | 위치 |
|------------------|------|
| 카드 표시 시간 | `app.js`의 `RotationEngine.DEFAULT_INTERVAL_MS` (기본 8000ms) |
| 페이드 전환 시간 | `app.js`의 `TransitionManager.DEFAULT_CONFIG` (각 500ms) |
| 폰트 크기 | `style.css`의 `--font-size-main`, `--font-size-sub` |
| 카드 배경 팔레트 | `app.js`의 `StyleManager.CARD_STYLES` |
| 헤더 타이틀 | `index.html`의 `.brand-header__title` 텍스트 |

## 디자인 원칙

- **단순성**: 빌드 과정 없이 정적 파일만으로 동작
- **안정성**: 8시간+ 연속 실행을 위한 메모리 관리와 에러 격리
- **유연성**: 텍스트 파일 수정만으로 운영자가 직접 문구 변경 가능
- **시각적 완성도**: AWS 브랜드 컬러 + 한글 단어 단위 줄바꿈(`word-break: keep-all`)

## 운영 팁

- 행사 시작 전 풀스크린 모드로 띄워두고, TV 절전 모드/화면보호기를 비활성화한다
- 노트북으로 띄우는 경우 절전 모드 진입을 막아둔다 (macOS: `caffeinate`)
- TV 옆 임시 노트북에서 `messages.txt`를 편집하면 별도 배포 없이 문구가 즉시 갱신된다
